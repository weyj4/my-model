## Active Issues / Pain Points

### Tokenizer environment breakage
The HF fast tokenizer (`tokenizers` Rust backend) returns empty tensors in the RunPod PyTorch container and on M2 Mac with `transformers==5.3.0`. Root cause: version mismatch between `transformers` and `tokenizers` libraries. Workaround is to always use `tiktoken` directly. Will not affect SFT (trl uses tokenizers internally in a proper environment).

### pretokenize.py dataset is hardcoded
`scripts/pretokenize.py` currently hardcodes `"HuggingFaceFW/fineweb"`. For the FineWeb-Edu run, the dataset name was changed directly. This should be refactored into a `--dataset` argument (see Planned Refactors below).

### Critical bug in prepare_fineweb_edu.py (now superseded)
The script written during the session used `all_tokens = []` (Python list) to accumulate 3.3B tokens. This would require ~92GB RAM due to Python integer overhead, and converts to numpy only at the end (requiring a second full copy). The correct approach ‚Äî used in `pretokenize.py` ‚Äî is to pre-allocate a numpy array upfront and write into it in slices. `prepare_fineweb_edu.py` is now superseded by using `pretokenize.py` with `--dataset fineweb-edu`.

### No LR warmup or cosine decay (current runs)
`warmup_steps=100` in `TrainingConfig` exists but is **not wired into the optimizer** ‚Äî it's a dead config field. The optimizer uses a fixed LR throughout. This leaves measurable loss improvement on the table in the final third of training. Fix is ready (see Next Steps).

### out_head.bias dropped in HF conversion
Your model has `out_head.bias [50257]` but standard GPT2LMHeadModel doesn't support `lm_head.bias`. It was dropped during conversion rather than injected. Minor quality loss. Not worth fixing unless you retrain with `bias=False` in the output head.

## Planned Refactors

### Priority 1 ‚Äî Wire up LR scheduler in `train.py`

Add to `config.py`:
```python
min_lr: float = 4e-5          # lr / 10
lr_decay_steps: int = 49743   # set via CLI arg in launch script
warmup_steps: int = 2000      # increase from 100
```

Add `get_lr()` function before `train()`:
```python
def get_lr(step, train_cfg):
    if step < train_cfg.warmup_steps:
        return train_cfg.lr * (step + 1) / train_cfg.warmup_steps
    if step > train_cfg.lr_decay_steps:
        return train_cfg.min_lr
    ratio = (step - train_cfg.warmup_steps) / (train_cfg.lr_decay_steps - train_cfg.warmup_steps)
    coeff = 0.5 * (1.0 + math.cos(math.pi * ratio))
    return train_cfg.min_lr + coeff * (train_cfg.lr - train_cfg.min_lr)
```

Add at top of inner training loop:
```python
lr = get_lr(global_step, train_cfg)
for param_group in optimizer.param_groups:
    param_group['lr'] = lr
```

Fix AdamW betas:
```python
optimizer = torch.optim.AdamW(..., betas=(0.9, 0.95), ...)
```

Add `"train/lr": lr` to wandb.log dict.

### Priority 4 ‚Äî Separate weight decay param groups in `train.py`

```python
decay_params = [p for n, p in model.named_parameters() if p.dim() >= 2]
nodecay_params = [p for n, p in model.named_parameters() if p.dim() < 2]
optim_groups = [
    {"params": decay_params, "weight_decay": train_cfg.weight_decay},
    {"params": nodecay_params, "weight_decay": 0.0},
]
optimizer = torch.optim.AdamW(optim_groups, lr=train_cfg.lr, betas=(0.9, 0.95))
```

This stops weight decay from being applied to LayerNorm scale/shift and biases.

## Architecture Reference (quick lookup)

```
GPTConfig defaults:
  vocab_size=50257, context_length=1024, emb_dim=768
  n_heads=12, n_layers=12, drop_rate=0.0, use_flash=True

Parameter count: 163M (bias=True on all Linear layers)
Chinchilla optimal tokens: 20 √ó 163M = 3.26B
tokens/step at batch=64: 64 √ó 1024 = 65,536
Steps to Chinchilla optimal: 3,260,000,000 / 65,536 = 49,743

Checkpoint structure:
  ckpt["step"]                  int
  ckpt["model_state_dict"]      dict ‚Äî bf16, _orig_mod. prefixed (torch.compile)
  ckpt["optimizer_state_dict"]  dict ‚Äî fp32
  ckpt["config"]                dict ‚Äî GPTConfig as dict

HF weight key remapping notes:
  - Strip _orig_mod. prefix before remapping
  - norm1.scale ‚Üí ln_1.weight, norm1.shift ‚Üí ln_1.bias
  - W_query/key/value ‚Üí concatenate then .T ‚Üí c_attn.weight [768, 2304]
  - ff.layers.0.weight ‚Üí .T ‚Üí c_fc.weight [768, 3072]
  - ff.layers.2.weight ‚Üí .T ‚Üí c_proj.weight [3072, 768]
  - All biases: no transpose
  - out_head.weight: no transpose (not Conv1D)
```
